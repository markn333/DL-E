# Claude Code 開発憲章 v1.7.0

> 対象: C / C++ / C#　チーム規模: 大規模（21名～）

## 基本方針

- Claude Code の提案はあくまで**候補**。最終判断は常に開発者が行う
- 生成コードは**必ずレビューを経てからコミット**する
- **セキュリティ・安全性・可読性・保守性**を常にパフォーマンスより優先する
- 本憲章のルールはチーム全員が遵守する

## 詳細ルールの参照先

| ルール | ファイル | 適用タイミング |
|--------|----------|----------------|
| 共通コーディング規約・複雑性指標 | `.claude/instructions/coding-common.md` | 全ファイル編集時 |
| C / C++ 固有規約 | `.claude/instructions/c-cpp.md` | .c/.cpp/.h/.hpp 編集時 |
| C# 固有規約 | `.claude/instructions/csharp.md` | .cs 編集時 |
| セキュリティ規約 | `.claude/instructions/security.md` | 全ファイル編集時 |
| レビュー・チェックリスト | `/review` コマンド | レビュー時に呼ぶ |
| 試験項目作成 | `/test` コマンド | テスト作成時に呼ぶ |
| 波及影響確認 | `/impact` コマンド | 修正後に呼ぶ |

## バグ修正時の記録ルール

バグを修正した場合、必ず `bugs/` ディレクトリに記録を残すこと。

- ファイル: `bugs/BUG-NNNN.md`（NNNN は既存ファイル数 + 1 をゼロ埋め4桁）
- bugs/ ディレクトリがなければ作成する
- 以下のフォーマットで記録する:

```markdown
# BUG-NNNN: [タイトル]
- 報告日: [YYYY-MM-DD]
- ステータス: 完了
- 重要度: 高 / 中 / 低

## 現象
[どのような問題が発生したか]

## 原因
[なぜ発生したか]

## 処置内容
[どのように修正したか、変更したファイル]

## 再発防止策
[同様の問題を防ぐための知見]
```

## 長時間バッチ処理のルール

長時間実行するバッチ処理（スクレイピング・データ生成・Embedding生成等）を作成する場合、以下の**4機能を必ず実装**すること:

| 機能 | 必須 | 実装例 |
|------|------|--------|
| **起動** | ✅ | `xxx_generate.bat` / `python xxx.py` |
| **停止** | ✅ | `xxx_stop.bat`（プロセスを安全に停止） |
| **再開** | ✅ | `skip_existing` / `progress.json` で途中から再開可能にする |
| **進捗確認** | ✅ | `xxx_status.bat` / `python xxx.py --stats`（件数・進捗率・ETA表示） |

タスクスケジューラや cron に登録した場合、**全件処理完了時にタスク登録を自動削除**すること（`schtasks /delete` 等）。

## Claude Code への絶対禁止事項

以下を Claude Code に**絶対に入力しない**こと：
APIキー・パスワード・トークン / 本番DB接続文字列・サーバIP / 個人情報 / 社内設計仕様書・顧客データ / 社内ライセンスキー
