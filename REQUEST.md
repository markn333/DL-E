# REQUEST: DL-E（ディープラーニングG検定 模擬試験アプリ）

作成日: 2026-04-23

## 概要

書籍『徹底攻略 ディープラーニングG検定ジェネラリスト問題集 第3版』の問題を元にした、ディープラーニングG検定試験対策の模擬試験 Web アプリを作成する。
PWA としてオフライン動作・iOS/Android 対応とし、最終的に GitHub Pages 等で Web 公開する。

## 背景・動機

- ディープラーニングG検定ジェネラリスト試験対策のため、書籍の問題を効率よく繰り返し解ける学習環境が欲しい
- 通勤・隙間時間を含めスマホで学習したいので PWA（オフライン）化が必須
- OSSDB プロジェクトで同種アプリの雛形が完成しているため、それを再利用し DL-E 用に書き換える

## 要件

### 機能要件
- PDF（書籍）をパースして Markdown / JSON 等の使いやすい形式に変換すること
- 問題数は 10問 / 20問 / 50問 / 100問 から選択できること
- 出題範囲（カテゴリ／章）を選択できること
- 出題モード: ランダム / 苦手問題 / 順番通り
- 試験を中断・再開できること（LocalStorage 等）
- 正解率の履歴をグラフで確認できること
- 不正解の問題を記録し、弱点の傾向を分析できること
- 不正解のみ再挑戦できること
- 解説表示・カテゴリ別正解率表示
- PWA としてオフラインで動作すること
- iOS / Android（モバイルブラウザ）で動作すること
- 最終形は GitHub に Web 公開し、インターネットから利用できること

### 非機能要件
- ビルドツール不要（純粋な HTML/CSS/Vanilla JS）でメンテしやすいこと
- LocalStorage の容量に収まる範囲で問題データを管理
- Service Worker でオフラインキャッシュ
- レスポンシブ対応（スマホ縦持ち優先）

## 技術スタック

- 言語: HTML5 / CSS3 / Vanilla JavaScript
- フレームワーク: なし（PWA / Service Worker）
- ライブラリ: Chart.js（履歴・弱点分析グラフ）
- データ形式: questions.json（id / chapter / category / question / choices / answer / explanation / multiAnswer）
- PDF→MD 変換: pdf2md など既存ツール（要選定）/ 手動補正
- 動作環境: モバイル/PC ブラウザ、ローカルファイル/HTTP サーバー、最終的に GitHub Pages

## 前提条件

- ベースとして OSSDB 問題集アプリ（同 Mothership 配下 `projects/OSSDB`）の構造・JS 一式を流用する
- 書籍 PDF（徹底攻略シリーズ）はプロジェクト直下に配置済み（150MB）
- GitHub Pages へ公開可能なアカウントがある

## 制約事項

- 書籍の著作権を侵害しない範囲で問題データを利用する（公開リポジトリに問題本文をそのまま含めるかは要検討）
- PDF（150MB）は Git 管理対象から除外する想定（.gitignore）
- LocalStorage の上限（5〜10MB 目安）に収まるように questions.json を管理する
